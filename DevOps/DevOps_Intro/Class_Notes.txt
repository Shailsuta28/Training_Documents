			We can able to achieve:
			
				- Continuous Development
				
				- Continuous Integration 
				
				- Continuous Testing 
				
				- Continuous Delivery
					- It is the process to release the changes to production environment.
					- Expects the Manual Approvals 
				
			We cannot achieve :
			
				- Continuous Deployment 
					- It is the process to release the changes to production environment.
					- It Never expect any Manual intervension/Approvals 
					
					
	- DevOps ::::
	
		- DevOps is a Software Development Strategy/Process, which helps to promote the collaboration between the teams like Development Team and the Operations Team to achieve Continuous Development, Continuous Integration, Continuous Testing, Continuous Delivery, Continuous Deployment and Continuous Monitoring in a more automated fashion.
		
	
	- How to Implement DevOps? :::
	
		- Teams:
		
			- DevOps Team 
				- Infra-Structure Management Team - Create Servers/Configure Servers - 200 Test Servers 
				- Application Development Team 
				- Testing Team 
				- Release Management Team 
				- Production Support Team 
				- Production Monitoring Team 
				- IT Security Team

		- Application Architecture :	Application Owners/Architects

			- Monolith Application Architecture :
			
				- These are Tightly Coupled Applications
				- All the Functions/Modules and Dependencies are Tightly coupled
				- With this, we can achieve just Continuous Delivery. We cannot achieve Continuous Deployment
			
			
			- Micro-Service Based Application Architecture :
			
				- These are loosely Coupled Application Architecture
				- Each and every function is considered as a Micro-Service
				- Each Micro-Service can be independenly development and deployed to production
				- With this, we can achieve Continuous Deployment

				
				
				
		- DevOps Stages:
		
			- Continuous Development :::
			
				- It is the capability of any Application Development Team to Continuously Develop code changes.
				- It is a process that is used to improve the Developers' productivity.
				
				Role of the Developer :
				
					- Understand the Requirement and Create the Source Code - Java/.Net/Python				
					
					- Created the Source Code :
					
				Manual Activities :::
				
					- Build Application
						- Process to compile and create the artifacts(Binaries - *.war/*.jar/*.exec/*.dll
					- Unit Testing
					- Promote the Changes for further Testing Environments
					- Notify the Testing Team thru emails
				
				Using DevOps Approach :::
				
					Automate :::
						- Build Application
							- Process to compile and create the artifacts(Binaries - *.war/*.jar/*.exec/*.dll
						- Unit Testing
						- Promote the Changes for further Testing Environments
						- Notify the Testing Team thru emails	

					- Created the Source Code  - Java/.Net/Python
					- Save the Source Code in the Source Code Repositories(GitHub)
					
				Tools ::
				
					- IDEs - Integrated Development Environments(Eclipse, Pycharm, IntelliJ, Visual Studio ,Visual Studio Code)
					- GIT, GITHUB Repositories
					- Build Tools, Unit Testing 
					
			
			- Continuous Integration ::: 
			
				- It is the Capability of the Application Development Team to Continuously integrate their changes for further testing.
			
				Automate :::
						- Build Application
							- Process to compile and create the artifacts(Binaries - *.war/*.jar/*.exec/*.dll)
						- Unit Testing
						- Promote the Changes for further Testing Environments
						- Notify the Testing Team thru emails	
	

				- Tools :
					- IDEs - Integrated Development Environments(Eclipse, Pycharm, IntelliJ, Visual Studio ,Visual Studio Code)
					- GIT, GITHUB Repositories
					- Build Tools, Unit Testing
					- Docker, Ansible, Kubernetes 
					- Jenkins
					
			
			- Continuous Testing :::
			
				- It is the capability of the Testing Team to Continuously Test the Changes 
				- Test Cases/Test Scenarios - Test Reports 
				
				- Tools :
				
					- TestNG, Selenium
					- Jenkins
